Feature: Lemonade Website Smoke Test

  Scenario: User lands on the homepage and navigates to Renters Insurance

📁 File: features/step-definitions/commonSteps.js

javascript
const { Given, When, Then } = require('@wdio/cucumber-framework');
const assert = require('assert');

Given('I open the Lemonade homepage', async () => {
  await browser.url('https://www.lemonade.com');
  await browser.pause(1000); // Let it load fully
});

Then('I should see the logo and call-to-action', async () => {
  const logo = await $('img[alt="Lemonade"]');
  const cta = await $('a=Check Our Prices');

  assert(await logo.isDisplayed(), 'Logo is not visible');
  assert(await cta.isDisplayed(), 'CTA button is not visible');
});

When('I click on the "Renters" link in the header', async () => {
  const rentersLink = await $('header a[href="/renters"]');
  await rentersLink.click();
  await browser.pause(1000);
});

Then('I should be redirected to the Renters Insurance page', async () => {
  const url = await browser.getUrl();
  assert(url.includes('/renters'), 'Did not navigate to Renters page');

  const heading = await $('h1');
  const headingText = await heading.getText();
  assert(
    headingText.toLowerCase().includes('renters'),
    'Renters heading not found'
  );
});

Yes! If you want to push code to GitHub without logging in every time, you have two main options:

✅ Option 1: Use a Personal Access Token (recommended by GitHub)

Since GitHub no longer supports password authentication, you can use a Personal Access Token (PAT) instead of your password when pushing.

💡 Quick Steps:

🔐 Step 1: Create a Personal Access Token

  1. Go to https://github.com/settings/tokens

  2. Click "Generate new token" → "Fine-grained token"

  3. Select:

    • Repository access: Only select repositories or All repositories

    • Permissions: Under "Repository", enable Read and Write

  4. Click Generate token

  5. Copy the token (you won’t see it again!)

💻 Step 2: Push your code using the token

When you run:

bash
git push -u origin main

You’ll be asked for your GitHub username and then your password — paste the token instead of a password.

💡 Tip: To avoid entering the token every time:

bash
git config --global credential.helper store

This saves your token locally after first use.

✅ Option 2: Use SSH Authentication (login-free forever 🎉)

This is a one-time setup, but very convenient once done.

🔐 Step 1: Generate SSH Key

bash
ssh-keygen -t ed25519 -C "your_email@example.com"

  • Hit Enter to accept the default location

  • Enter a passphrase (optional)

📋 Step 2: Add your SSH key to GitHub

  1. Copy your key:

bash
cat ~/.ssh/id_ed25519.pub

  1. Go to https://github.com/settings/keys

  2. Click "New SSH Key", give it a name, and paste your key

🔗 Step 3: Change your repo remote to use SSH

bash
git remote set-url origin git@github.com:yourusername/lemonade-qa-task.git

Then push like usual:

bash
git push -u origin main

No username/token needed again.
